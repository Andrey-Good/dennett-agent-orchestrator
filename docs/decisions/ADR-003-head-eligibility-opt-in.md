# ADR-003: Head eligibility выдаётся только явно

- **Статус:** принят
- **Дата:** 2026-07-13

## Контекст

Автоматическое повышение любого trusted device может раскрыть owner keys, создать split-brain или неожиданно превратить laptop/phone в сервер.

## Решение

Все устройства начинают с `head_eligibility = none`. Владелец может выдать:

- `emergency` — ограниченное локальное продолжение работы;
- `full` — возможность получить обычную Head authority после проверки свежести, ключей, canonical data, fencing и health.

Изменение требует сильного step-up authentication. Models, projects и импортированные packages не могут выдать эту роль. Автоматический failover использует только заранее разрешённых `full`-кандидатов.

## Последствия

Failover становится предсказуемым. UI обязан показывать роль, readiness и отсутствующие prerequisites; наличие локального кэша само по себе ничего не разрешает.
