## Stage 11 Decision

Stage 11 proves a controlled local package artifact path only:

- the artifact source is a local `.tgz` produced by `npm pack --json --ignore-scripts` from the current checkout after `pnpm build`;
- the consumer path is a temporary local npm project that installs the `.tgz` with `npm install --ignore-scripts --no-audit --fund=false`;
- the installed command is smoked through the package `bin` entry by running `dennett-agent-orchestrator --help`;
- package uninstall removes the installed package directory and generated package bin from that temporary consumer project;
- upgrade and rollback are proven only when the caller supplies two distinct existing `.tgz` artifacts with `--from-tgz` and `--to-tgz`;
- local SBOM proof validates an SPDX SBOM generated by `npm sbom` from the temporary consumer install.

The package is not published even though `package.json` now uses the prepared `0.1.0-rc.1` release-candidate version and no longer carries the npm `private` guard. Stage 11 does not run `npm publish`, create tags, push commits, create GitHub releases, sign artifacts, emit provenance, or retain a canonical SBOM artifact.

## Evidence Matrix

| Evidence area | Command or owner | Proven result | Limit |
| --- | --- | --- | --- |
| Generated distribution shape | `pnpm dist:check` | `dist/src/interfaces/cli.js` exists, forbidden `dist/vitest.config.*` output is absent, and CLI help contains the command identity. | Requires `pnpm build`; `dist` is build-local and not tracked source. |
| Package inventory and metadata | `pnpm packlist:check` | Package contents are allowlisted to package metadata, README, LICENSE, built CLI files, and JSON schemas; version is the prepared `0.1.0-rc.1` release candidate; Node floor is `>=22.13.0`; repository, bugs, homepage, and keyword metadata match the launch-readiness allowlist. | Does not prove public registry availability, registry ownership, publication approval, or public prerelease publication. |
| Local install and uninstall | `pnpm package:local-install:proof` | Builds and packs the current checkout, installs the resulting `.tgz` into a temporary npm consumer project, smokes the installed bin, uninstalls the package, and verifies package/bin removal. | Proves package removal from a temporary consumer project only, not local app-state or provider-data deletion. |
| Upgrade and rollback harness | `pnpm package:upgrade-rollback:proof -- --from-tgz <old.tgz> --to-tgz <new.tgz>` | Installs `from`, smokes CLI help, installs `to`, smokes CLI help, reinstalls `from`, and smokes CLI help again. | Requires two distinct existing tarballs. No prior artifact means rollback proof is unavailable. |
| Local SBOM proof | `pnpm supply-chain:local:proof` or `pnpm supply-chain:local:proof -- --from-tgz <candidate.tgz>` | Installs a local tarball into a temporary consumer project and validates that `npm sbom --sbom-format=spdx --sbom-type=application` returns an SPDX document naming the package. | Does not retain a canonical SBOM file, attach an SBOM to a release, sign the SBOM, or publish it. |
| CI package proof job | `.github/workflows/ci.yml` `package-proof` job | Configures local install/uninstall proof on `ubuntu-latest`, `windows-latest`, and `macos-latest` with Node 24 and pnpm 10.33.0. | A support claim for an OS requires a green CI run for that matrix entry; this document alone is not live CI evidence. |

## Allowed Claims

The following claims are allowed when the corresponding command or CI run has passed:

- a locally packed `.tgz` artifact can be installed into a temporary npm consumer project;
- the installed package exposes a working `dennett-agent-orchestrator` bin that prints help;
- uninstall removes the installed package directory and package bin from that temporary consumer project;
- upgrade/rollback can be smoke-tested between two explicit local tarballs;
- an SPDX SBOM can be generated and validated locally from a temporary consumer install;
- CI is configured to run local package install/uninstall proof on Ubuntu, Windows, and macOS.

## Forbidden Claims

Do not claim:

- the package is publicly published to npm or any registry;
- npm namespace ownership is verified;
- `private: false` is approved;
- a public npm install path is available;
- a Git tag, pushed commit, GitHub release, or public release artifact exists because of Stage 11;
- install, uninstall, upgrade, rollback, SBOM, or package proof covers anything except the documented local tarball paths;
- uninstall removes local app state, SQLite state, workspace files, provider registry/config, memory provider data, runtime data, logs, or external provider records;
- Linux, Windows, or macOS are publicly supported without recorded green evidence for the specific claimed environment;
- artifacts, SBOMs, releases, or provenance are signed or retained;
- public package launch, general availability, production readiness, hosted operation, installer distribution, or container distribution is complete.
